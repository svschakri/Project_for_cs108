\section{Introduction}
This is a project made by us as part of the CS108 course (Spring 2026) which includes a mini game hub consisting of three two-player board games.
\\
This game hub includes the following features:
\begin{itemize}
    \item \textbf{Authentication:} The code for this is covered in \texttt{main.sh}. \\ For starting the game, the players enter their username and password and login (If username doesn't exist they register). \\ After logging in, the game interface shows up for the games to start between the players.
    \item \textbf{Game Engine: } The code for this is covered in \texttt{game.py}. \\ This includes the main game interface as well as other menus like game over options.
    \item \textbf{Games:} Included games are modified versions of Tic-Tac-Toe, Othello and Connect Four.
\end{itemize}
\subsection{Description of Games:}
\begin{itemize}
    \item \textbf{Tic-Tac-Toe (Expanded):} This is a modified version of tic-tac-toe which has a 10 x 10 board with two players playing X or O on the board.
    \\ It requires the player to have 5 pieces in a horizontal, vertical or diagonal (consecutive) alignment to win.
    \\ The game ends in a draw if the board gets filled and no player has 5 of their pieces aligned.
    \item \textbf{Othello (Reversi):} This is the original Othello game, played on an 8 x 8 board with two black discs and two white discs in the center.
    \\ A move is valid only if it traps one or more opponent discs in a straight line between the newly placed disc and an existing own disc. If a player has no valid move then their turn is skipped.
    \\ The player with more discs of their colour when no valid moves remain wins.
    \item \textbf{Connect Four: } This game is played on a 7 x 7 board. Each player has his own designated coin and coins fall into the lowest row of each column. 
    \\ A player wins by getting 4 of their coins aligned diagonally, horizontally or vertically consecutively. The game ends in a draw if the board fills with no winner.
\end{itemize}

This Project emphasizes of design of these games as well as addition of some extra features.
\section{Running instructions}
\begin{itemize}
    \item The game requires python version 3.11 or above along with pip.
    \item Clone the project repository on your local device and move to the root directory in the terminal.
    \item \textbf{Optional:} Setup a virtual environment for installing dependencies as follows:
    \begin{lstlisting}[frame=single]
        python3 -m venv .venv
    \end{lstlisting}

    Activate the virtual environment in the terminal by:
    \begin{lstlisting}[frame=single]
        source .venv/bin/activate
    \end{lstlisting}

    \item Install dependencies from requirements.txt
    \begin{lstlisting}[frame=single]
        pip install -r requirements.txt
    \end{lstlisting}

    \item Before starting the game run:
    \begin{lstlisting}[frame=single]
        bash ./reset.sh
    \end{lstlisting}
    This resets the game and forms the necessary files for the game to start.
    \\
    \textbf{**CAUTION** : Don't run reset.sh carelessly as it resets the game history.}
    \item Start the game hub by running the following command on the terminal 
    \begin{lstlisting}[frame=single]
        bash ./main.sh 
    \end{lstlisting}
    \item  Follow the instructions on the screen from now on.
    \item To compile the report run:
    \begin{lstlisting}[frame=single]
        make
    \end{lstlisting}
    \item After running \texttt{make}, always run the following to clear auxiliary files generated:
    \begin{lstlisting}[frame=single]
        make clean
    \end{lstlisting}
\end{itemize}

\subsection{Game Engine}
The code for the game engine is included in game.py. This includes the start menu, as well as code for game over menu and the leaderboard screen. \textbf{AI generated images} have been used for the theme of \textbf{Hollow Knight} game \cite{hollowKnight}.

The start menu on start of a game between Bipul and Chakri will look as follows:

    \caption{Authenticator window in terminal}
    \label{fig:start_menu}

The users can then go on to click on the icons of the games to enter the game interface.
\\
\texttt{game.py} essentially has the \texttt{Player}, \texttt{Board} and \texttt{Game} class which would contain the necessary functions for the games to work as this class would be inherited by the other game files.
\\
The \texttt{Game} class constructor includes \texttt{Player} and \texttt{Board} objects, for storing player information and board state at each move (as a numpy array).
\\
\\
Some of the important functions in the \texttt{Game} class are:
\begin{description}
    \item[\texttt{Game.check\_win()}]
    - Overridable function to check win condition in each game.

    \item[\texttt{Game.switch\_turn()}]
    - Switches turn of players after each move in every game.

    \item[\texttt{Game.draw\_game\_over()}]
    - Draws the game over screen after each game ends.

    \item[\texttt{Game.update\_result()}]
    - Updates result of each game based on the result of the game into \texttt{history.csv} which saves game name, winner, loser and date of playing of game.

    \item[\texttt{Game.reset\_game()}]
    - Resets game, this can be called during play to reset the board (overriden in othello.py).

    \item[\texttt{Game.back\_game()}]
    - Reverts the last move played in the game (overriden in othello.py).
\end{description}

Some of the other functions used in game.py are as follows:
\begin{description}
    \item[\texttt{play\_game()}] - Runs \texttt{games/*.py} files (game files) as a module using \texttt{\_\_import\_\_} inbuilt method.
    \item[\texttt{analysis\_menu()}] - Shows leaderboard after each game by calling \texttt{leaderboard.sh} using \texttt{subprocess} module. Also takes input for the user (through a GUI) for how the leaderboard data to be displayed is to be sorted.
    \\
    Additionally used for making analysis window using \texttt{matplotlib} module.
    \item[\texttt{game\_loop()}] - Runs the main game loop.
    \item[\texttt{start\_menu()}] - Runs GUI interface for the start menu.
\end{description}

Further in each of the game files, there is a \texttt{run()} function which is called by \texttt{game.py} in \texttt{play\_game()} function to run the games.

\subsection{Leaderboard and Analytics}
Leaderboard is created using \texttt{history.csv} which gets updated everytime when a game ends in the hub.
\\
\texttt{history.csv} is in the following format:
\\
\textbf{Game,Draw,Winner,Loser,Date}
\\
\\
Here the columns denote the following:
\begin{description}
    \item[Game] - Contains game name
    \item[Draw] - Is empty if no draw is made, otherwise it stores "Draw".
    \item[Winner] - Contains the winner if there is no draw, otherwise holds the name of player 1.
    \item[Loser] - Contains the loser if there is no draw, otherwise holds the name of player 2.
    \item[Date] - Contains the date at which the game was played.
\end{description}

With the data contained in \texttt{history.csv}, leaderboard is created using \texttt{leaderboard.sh}. Some of the features of this script are as follows:
\begin{itemize}
    \item Uses awk \cite{awkDocs} to read the csv file line by line and create associative arrays having wins, losses and win-loss ratio.
    \item Output given from awk is formatted based on "Sort by" input given by the user. The leaderboard can be sorted based on --- \textbf{wins}, \textbf{losses} or \textbf{win-loss ratio}.
    \item Sort formatting is done using \texttt{sort} command in bash.
\end{itemize}

\section{Modules Used}

\section{Features and Additional Improvements}
\begin{itemize}
    \item \textbf{Use of AI generated images for design:}
    \\
    AI generated images have been used to make the project look better. The use of these images was definitely challenging. This involves a good amount of prompting knowledge as well as some design skills.
    \item \textbf{Added back and reset features in all games:}
    \\
    This feature has been added to make the project more user friendly and less frustrating for the players. It was challenging to add these features especially in Othello as it required to think of an algorithm to store and use the last flipped coins.
    \item \textbf{Adding game sprites for characters:}
    \\
    This involved adding game sprites which made the game more interacting and fun to play, e.g., adding happy player sprite next to the winning player after a game. Since, this was made from AI generated images, It was challenging to locate and resize the images on the screen.
\end{itemize}
